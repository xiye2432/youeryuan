# 一、厨房工作台 UI 改造

## 1. `taro-app1/src/pages/login/index.tsx` — 厨房角色登录后跳转厨房工作台

### 改动位置：第113-117行，`onLoginSuccess` 中的跳转逻辑

**原代码：**
```tsx
safeToast('登录成功', 'success')
setTimeout(() => {
  safeGo('/pages/index/index')
}, 1000)
```

**改为：**
```tsx
safeToast('登录成功', 'success')
setTimeout(() => {
  const role = user.role || 'TEACHER'
  if (role === 'KITCHEN') {
    Taro.redirectTo({ url: '/pages/kitchen/index' })
  } else {
    safeGo('/pages/index/index')
  }
}, 1000)
```

**说明：** 厨房人员登录后直接跳转到厨房工作台页面，不经过通用首页。

## 2. `taro-app1/src/pages/index/index.tsx` — 首页检测厨房角色自动重定向

### 改动位置：第41-49行，`useEffect` 和 `useDidShow` 中增加角色检测

**原代码：**
```tsx
useEffect(() => {
  void loadData()
  loadUser()
}, [])

useDidShow(() => {
  void loadData()
  loadUser()
})
```

**改为：**
```tsx
useEffect(() => {
  void loadData()
  loadUser()
  checkKitchenRedirect()
}, [])

useDidShow(() => {
  void loadData()
  loadUser()
  checkKitchenRedirect()
})

const checkKitchenRedirect = () => {
  const user = Taro.getStorageSync('kt_current_user')
  if (user?.role === 'KITCHEN') {
    Taro.redirectTo({ url: '/pages/kitchen/index' })
  }
}
```

**说明：** 防止厨房人员通过其他途径进入通用首页时看到无关内容，自动重定向到厨房工作台。

## 3. `taro-app1/src/pages/kitchen/index.tsx` — 重写为厨房工作台

### 改动类型：完整重写

**主要变更：**

1. **新增 `viewMode` 状态**：从 `'current' | 'history'` 扩展为 `'dashboard' | 'current' | 'history'`，厨房角色默认进入 `dashboard`
2. **新增辅助函数：**
   - `getTodayDayIndex()` — 根据星期几获取当日索引
   - `getTodayMeals()` — 获取今日菜谱数据
   - `getDraftRecords()` — 获取草稿食谱列表
   - `getDietRiskCount()` — 获取待处理食谱数量
   - `getTodayPriorityText()` — 生成今日优先提醒文案
   - `getDishName()` — 通用提取菜品名称（含午餐多菜品合并）
3. **新增 `renderDashboard()` 厨房工作台视图**，包含：
   - 顶部问候 + 角色标识（🍳 厨房）
   - 今日重点提示横幅（🔔 根据食谱状态动态生成）
   - 三个关键指标卡片：食谱状态、待处理食谱数、今日用餐人数
   - 今日菜单摘要（7个餐次网格预览）
   - 待发布食谱提醒（草稿食谱一键发布）
   - 过敏与忌口提醒摘要
   - 本周食谱摘要（周次、园区、状态、营养数据）
   - 快捷入口：当前食谱、历史食谱、发布食谱、同步数据
4. **新增 `publishRecord()` 函数** — 厨房可直接发布草稿食谱
5. **`renderRecipeView()`** 保留原有食谱查看/编辑功能，厨房角色增加"← 工作台"返回按钮
6. **条件渲染**：`isKitchen && viewMode === 'dashboard'` 时显示工作台，否则显示食谱查看

## 6. `taro-app1/src/pages/students/attendance.tsx` — 批量考勤默认全班全勤

### 改动类型：批量考勤逻辑调整

**本次修改内容：**

1. 批量考勤页面打开后，自动为当前班级/当前范围内学生默认生成"出勤"记录。
2. 保留手动切换状态能力，支持将学生改为：
   - 缺勤
   - 病假
   - 其他
3. 当选择"其他"时，页面会显示备注输入框，支持填写具体说明。
4. 原"全部出勤"按钮改为"恢复全勤"，方便老师一键恢复默认全勤状态。
5. 统计栏同步调整为：出勤 / 缺勤 / 病假 / 其他 / 未记录。
6. 统计时仅计算当前页面学生范围，避免被其他班级数据干扰。

**说明：** 这次修改只针对"班级批量考勤登记"流程，满足"默认自动打钩全班全勤，手动选择缺勤、病假，以及其他（留备注）"的需求。
